# Анализ прошедших матчей в DOTA 2

## Обзор проекта
**Цель проекта:** Создать централизованный аналитический инструмент для анализа статистики матчей DOTA 2, который позволит разработчикам и аналитикам игры принимать обоснованные решения по балансу игрового процесса, развитию контента и улучшению пользовательского опыта.

**Задачи:**
*   Собрать и визуализировать ключевые метрики игры для общего мониторинга.
*   Проанализировать баланс между фракциями (Radiant/Dire) и типами атаки героев.
*   Выявить самые популярные герои и предметы в текущей мете.
*   Предоставить инструмент для детального анализа матчей по различным срезам.
*   Сформировать data-driven рекомендации для разработчиков.

**Целевая аудитория:** Гейм-дизайнеры, баланс-менеджеры, аналитики игровой индустрии.

**Стек технологий:** `Yandex DataLens`.

##  Ключевые выводы и инсайты

### 1. Масштабы и активность
*   Проанализировано **1802 матча** с участием **868 уникальных игроков** и **124 уникальных героев**.
*   **Баланс фракций практически идеален:** Распределение побед составляет **51%/49%** в пользу Radiant. Такой незначительный перевес (в пределах статистической погрешности) свидетельствует об **отличной сбалансированности карты и стартовых условий**, что является ключевым фактором для честной конкурентной среды.
*   **Сбалансированность типов урона:** Распределение игроков по типам атаки также близко к равновесию (**52%/48%**) в пользу Ranged. Это указывает на **здоровое разнообразие меты** — игроки не вынуждены выбирать героев одного типа для достижения успеха, что положительно влияет на вариативность игрового процесса.

### 2. Здоровое распределение продолжительности игр
*   **Нормальное распределение длительности матчей** с пиком на **30 минутах** является **индикатором стабильного игрового дизайна**. Большинство матчей достигают кульминации в предполагаемое разработчиками время, что говорит:
    *   Об отсутствии критических дисбалансов, приводящих к досрочным капитуляциям («снежному кому»).
    *   Об эффективности механизмов «кэтчапа», позволяющих отыгрываться.
    *   О правильно настроенной шкале силы героев и построек на разных стадиях игры.

### 3. Мета-анализ выявляет векторы развития
*   **Топ-10 героев** по частоте выбора показывает текущие предпочтения сообщества. Концентрация пика популярности на 2-3(Rubic, Luna, Tiny) героях может указывать на их **синергию с актуальной метой предметов** или воспринимаемую силу.
*   **Топ-10 предметов на первом слоте** четко определяет **стратегию ранней игры** текущего патча. Доминирование определенных предметов(Blink Dagger, Power Treads, Magic Wand) сигнализирует об их cost-efficiency и влияет на балансировку стартовых характеристик героев.

## Описание дашборда

Дашборд представляет собой комплексный аналитический инструмент, разделенный на смысловые блоки:

### **Панель KPI (общая статистика)**
*   Количество уникальных игроков
*   Общее число проанализированных матчей
*   Количество уникальных героев, участвовавших в матчах

###  **Анализ баланса**
*   **Соотношение побед по фракциям (Radiant/Dire):** Круговая диаграмма, наглядно показывающая баланс сил (51%/49%).
*   **Соотношение игроков по типу атаки:** Нормированная линейчатая диаграмма, отображающая баланс типов урона (52%/48%).

### **Статистика матчей**
*   **Распределение матчей по длительности:** Комбинированная диаграмма (гистограмма + кривая нормального распределения), наглядно демонстрирующая пик на 30 минутах.

### **Топ-листы (Мета-анализ)**
*   **Топ-10 персонажей по частоте выбора:** Столбчатая диаграмма, определяющая самых популярных героев.
*   **Топ-10 предметов на 1-м слоте экипировки:** Столбчатая диаграмма, показывающая ключевые стартовые предметы текущей меты.

### **Детализация по героям**
*   **Характеристики персонажей по ролям (Carry, Support, etc.):** Сводная таблица средних показателей побед и проигрышей в разрезе ролей.
*   **Сводная таблица игроков:** Детальная таблица, содержащая данные по игрокам (никнейм, выбранная фракция, количество матчей и доля побед).

### **Система фильтров**
Интерактивные фильтры для детального анализа:
*   **Фракция (Radiant/Dire)**
*   **Результат матча (Победа/Поражение)**
*   **Имя конкретного персонажа**
*   **Никнейм игрока**

**Все виджеты интерактивно связаны:** применение фильтра (например, выбор героя "Pudge") мгновенно пересчитывает KPI, топ предметов и распределение побед только для матчей с его участием.

## Рекомендации для разработчиков (Game Design & Balance)

1.  **Сохранять достигнутый баланс:** Текущие показатели (51/49 по фракциям, 52/48 по типам атаки) являются **целевым состоянием**. Основные усилия стоит направить на поддержание этого равновесия при выпуске нового контента (героев, предметов).

2.  **Сфокусироваться на углублении меты:**
    *   **Анализ аутсайдеров:** Изучить героев с низкой популярностью в топе. Их усиление может увеличить разнообразие выбора без нарушения общего баланса.
    *   **Балансировка через предметы:** Корректировка стоимости или характеристик предметов из **топ-10** — наиболее безопасный способ тонкой настройки силы популярных героев и типов урона.

3.  **Использовать 30-минутный маркер как эталон:** При введении новых механик или изменении старых оценивать их влияние на **среднюю продолжительность матча**. Существенные отклонения от 30-минутного пика могут быть индикатором disruptive-изменений.
