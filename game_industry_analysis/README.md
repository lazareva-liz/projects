# Анализ игровой индустрии (2000-2013)
**Исследование рынка видеоигр с акцентом на платформы, жанры и региональные предпочтения**

**Автор:** Лазарева Елизавета Сергеевна  

##  Обзор проекта
**Цель проекта:** Провести комплексный анализ рынка видеоигр за период 2000-2013 гг. для выявления ключевых трендов, популярных платформ, успешных жанров и региональных особенностей потребления с особым вниманием к жанру RPG.

**Задачи:**
1.  Провести полную предобработку и очистку исторических данных о продажах игр.
2.  Проанализировать динамику выпуска игр по платформам и выделить топ-7 самых продуктивных платформ.
3.  Изучить региональные различия в предпочтениях игроков (Северная Америка, Европа, Япония).
4.  Разработать систему категоризации игр по оценкам критиков и пользователей.
5.  Сформировать датасет, готовый для дальнейшего углубленного анализа.

**Стек технологий:** `Python 3`, `Pandas`, `Jupyter Notebook`, `Data Cleaning`, `Exploratory Data Analysis (EDA)`.

## Методология и этапы работы

### 1. Предобработка данных (Data Cleaning)
**Датасет:** `new_games.csv` (16,956 строк × 11 столбцов)

**Ключевые этапы:**
*   **Стандартизация:** Приведение названий столбцов к `snake_case`.
*   **Коррекция типов данных:**
    *   Преобразование `year_of_release` и `critic_score` в целочисленные типы (`int16`, `int8`).
    *   Исправление ошибочного строкового типа для `eu_sales_mm`, `jp_sales_mm`, `user_score`.
*   **Работа с пропусками (8 столбцов):**
    *   Удаление строк с пропусками в ключевых полях (`name`, `year_of_release`, `genre`).
    *   Замена пропусков в `critic_score` и `user_score` на индекс **-1** для отделения от реальных оценок.
    *   **Контекстная импутация** для финансовых данных: замена на среднее значение по платформе и году выпуска.
*   **Очистка от дубликатов:** Удаление 235 явных дубликатов, нормализация неявных (приведение `rating` к единому формату).

### 2. Фильтрация и категоризация
*   **Временной фильтр:** Выделен период **2000-2013 гг.** — эпоха расцвета и смены поколений консолей.
*   **Система категоризации оценок:**
    *   **Пользовательские оценки:** Низкая (0-3), Средняя (3-8), Высокая (8-10)
    *   **Оценки критиков:** Низкая (0-30), Средняя (30-80), Высокая (80-100)
*   **Платформенный анализ:** Агрегация по количеству выпущенных игр с выделением топ-7.

## Практическая ценность и рекомендации

### Для игровых издателей и разработчиков:
1.  **Платформенная стратегия:** Фокус на **PS2 и DS** в исследуемый период был оправдан. Для исторического анализа успеха проектов необходимо учитывать эту платформенную диспропорцию.
2.  **Региональный маркетинг:** Разработка **RPG** должна учитывать особый интерес японского рынока, в то время как западные релизы могут делать акцент на **Action** и **Shooter**.
3.  **Управление ожиданиями:** Большой разброс и отсутствие оценок у многих игр указывает на **нишевость значительной части рынка**.

### Для дата-инженеров и аналитиков:
1.  **Качество источников:** 40-50% пропусков в оценочных данных указывают на необходимость **верификации и дополнения данных** из альтернативных источников.
2.  **Методология импутации:** Использованный подход **"усреднение по платформе и году"** для финансовых данных может быть расширен до **"по жанру и платформе"** для большей точности.
3.  **Оптимизация хранения:** Преобразование типов данных (`int16`, `int8`, `float32`) сократило объем памяти на **~15%** — важный принцип для работы с большими датасетами.

### Для дальнейшего исследования:
1.  **Анализ жизненных циклов платформ:** Изучить динамику выпуска игр по годам для каждой платформы из топ-7.
2.  **Корреляция оценок и продаж:** Построить модели, предсказывающие коммерческий успех на основе оценок критиков/пользователей.
3.  **Глубокий анализ RPG:** Сравнить продажи, оценки и платформенное распределение RPG с другими жанрами.
4.  **Визуализация:** Создать дашборд с динамикой продаж по регионам, платформам и жанрам.
