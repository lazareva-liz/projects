
# Анализ инвестиций в IT-стартапы

**Цель проекта:** Провести комплексный анализ исторических данных о финансировании IT-стартапов за 35 лет для выявления закономерностей, которые могут помочь инвесторам в оценке перспективности проектов и построении стратегии инвестиций.

**Задачи:**
*   Провести предобработку и оценку качества исторических данных о стартапах за 35 лет.
*   Проанализировать паттерны финансирования: типичные суммы раундов, количество этапов привлечения инвестиций.
*   Исследовать взаимосвязь между характеристиками стартапа (категория, количество сотрудников) и его итоговым статусом (продан, закрыт, IPO).
*   Выявить аномалии в данных (символические сделки по $0-1) и предложить их бизнес-интерпретацию.
*   Сформулировать рекомендации для инвесторов на основе выявленных закономерностей.

**Стек технологий:** `Python`, `Pandas`, `NumPy`, `Matplotlib`, `Seaborn`, `Jupyter Notebook`, `Venn2`.

---

##  Ключевые выводы и бизнес-инсайты

1.  **Сделки по $0-1 — это стратегия, а не ошибка:** Анализ показал, что символические сделки (acqui-hire) чаще связаны либо с **продажей провальных проектов** с минимальным финансированием (команда как актив), либо с **спасением компаний от банкротства** с большими долгами. Это важный сигнал для инвесторов о реальной стоимости активов.

2.  **Категории-чемпионы по стоимости выходов:** Самые высокие медианные цены продажи у стартапов в категориях **`manufacturing`**, **`real_estate`** и **`automotive`**. Категория **`biotech`**, несмотря на высокий разброс цен, не вошла в топ, что указывает на высокие риски в этом сегменте.

3.  **Количество раундов как индикатор успеха:** Статистически подтверждена прямая связь между количеством привлеченных раундов и успешностью компании:
    *   **Купленные компании:** медиана = **0.5 раунда** (часто продаются до/после seed-раунда).
    *   **Закрытые/действующие компании:** чаще всего **1 раунд**.
    *   **Компании, вышедшие на IPO:** в среднем **~2 раунда** (1.98).

4.  **Риски ранних этапов:** Низкое количество раундов у закрытых стартапов подтверждает высокий **риск провала на ранних стадиях**. Инвесторам стоит тщательнее оценивать проекты на seed-этапе.

5.  **Качество данных как риск:** В данных обнаружены несоответствия (например, подозрительно низкое среднее число раундов у IPO-компаний). **40-52% записей о сотрудниках** не содержат данных об образовании, причем размер компании не влияет на полноту этих данных.

## Ход анализа и методы

1.  **Предобработка и оценка данных:**
    *   Объединение данных из 4+ таблиц по ключевым полям.
    *   Анализ пропусков, выбросов и аномалий. Построение воронок данных для оценки полноты.
    *   Принятие решений по очистке и консолидации.

2.  **Исследовательский анализ (EDA):**
    *   **Распределение статусов компаний:** анализ долей закрытых, действующих, купленных и вышедших на IPO стартапов.
    *   **Финансовые паттерны:** анализ типичных сумм раундов, цен продажи по категориям.
    *   **Корреляционный анализ:** связь между количеством раундов, суммой финансирования и итоговым статусом.
    *   **Анализ аномалий:** глубокий разбор сделок по $0-1 с бизнес-гипотезами.

3.  **Визуализация:**
    *   Гистограммы и боксплоты для распределений сумм.
    *   Столбчатые диаграммы для сравнения категорий.
    *   Диаграммы Венна для анализа пересечений данных.
    *   Точечные графики для выявления корреляций.

## Рекомендации для инвесторов и команды

### Для инвесторов (VC фонды, бизнес-ангелы)
*   **Фокус на manufacturing/real estate** для стратегии с предсказуемо высоким выходом.
*   **Осторожность с biotech:** высокий разброс результатов указывает на повышенный риск.
*   **Использовать "количество раундов" как ранний фильтр:** проекты, привлекшие 2+ раунда, статистически ближе к успешному выходу (IPO).
*   **Тщательная due diligence при сделках по $0-1:** оценивать реальные активы (патенты, команда) и обязательства.

### Для команды сбора данных (дата-инженеры)
*   **Уточнить методологию сбора данных по раундам:** расхождение между ожидаемыми (3-4 раунда до IPO) и фактическими (2 раунда) данными требует проверки.
*   **Внедрить валидацию на этапе сбора:** автоматические проверки на аномальные значения (цены, суммы раундов).
*   **Дополнить данные:** сбор информации о типе сделок (M&A, acqui-hire, IPO) для более точной классификации.

### Для дальнейшего анализа
*   Провести анализ временных рядов: влияние экономических кризисов на активность инвестиций и количество раундов.
*   Построить модель прогнозирования вероятности успешного выхода на основе признаков (категория, сумма первого раунда, количество сотрудников).
*   Сделать углубленный анализ "успешных неудач" — компаний с большим финансированием, но символической ценой продажи.
